Shared coding conventions make it possible for a team to collaborate efficiently. This rule allows to check compliance with a formatting standards. Default parameters value make this rule compliant with PSR2.

Quotes below are from the PSR-2 Coding Style Guide or the PHP: Language Reference - Manual.

"namespace" declarations should be followed by a blank line

Noncompliant Code Example

namespace Vendor\Package;
use FooClass;

Compliant Solution

namespace Vendor\Package;

use FooClass;

"use" declarations should be placed after "namespace" declarations

Noncompliant Code Example

use FooClass;

namespace Vendor\Package;

Compliant Solution

namespace Vendor\Package;

use FooClass;

"use" declarations should be follow by a blank line

Noncompliant Code Example

use FooClass;
$foo = 1;

Compliant Solution

use FooClass;

$foo = 1;

Open curly braces should be at the beginning of a new line for classes and functions

Noncompliant Code Example

class MyClass {
    function my_function(){
        ...
    }
}

Compliant Solution

class MyClass
{
    function my_function()
    {
        ...
    }
}

Open curly braces should be at the end of line for control structures

Noncompliant Code Example

if ($something)
{
    ...
}

Compliant Solution

if ($something) {
    ...
}
{code}

There should be exactly one space between closing parenthesis and opening curly braces

Noncompliant Code Example

if ($something)      {
    ...
}
{code}

Compliant Solution

if ($something) {
    ...
}

There should be exactly one space between control structure keyword and opening parenthesis or curly brace

Noncompliant Code Example

if($something) {
    ...
}

try{
    ...
} catch (Exception $e) {
    ...
}

Compliant Solution

if ($something) {
    ...
}

try {
    ...
} catch (Exception $e) {
    ...
}

There should not be any space after the opening parenthesis and before the closing parenthesis

Noncompliant Code Example

analyse( $fruit );

Compliant Solution

analyse($fruit);

There should be one space after each ";" in {{for}} statement

Noncompliant Code Example

for ($i = 0;$i < 10;   $i)
    ...
}

Compliant Solution

for ($i = 0; $i < 10; $i) {
    ...
}

Commas should be followed by one space and not preceded by any

Noncompliant Code Example

pressJuce($apply ,$orange);

Compliant Solution

pressJuce($apply, $orange);

There should not be any space after a method name

Noncompliant Code Example

do_something ();

Compliant Solution

do_something();

In foreach statement there should be one space before and after "as" keyword and "=>" operator.

Noncompliant Code Example

foreach ($fruits    as $fruit_key =>     $fruit) {
    ...
}

Compliant Solution

foreach ($fruits as $fruit_key => $fruit) {
    ...
}

Close curly brace and the next "else", "catch" and "finally" keywords should be located on the same line

Noncompliant Code Example

if ($something) {
    ...
}
else {
    ...
}

Compliant Solution

if ($something) {
    ...
} else {
    ...
}

List of implemented interfaces should be correctly indented

Lists of implements may be split across multiple lines, where each subsequent line is indented once. When doing so, the first item in the list MUST be on the next line, and there should be only one interface per line.

Noncompliant Code Example

class ClassName extends ParentClass implements \ArrayAccess, \Countable,
    \Serializable
{
}

Compliant Solution

class ClassName extends ParentClass implements
    \ArrayAccess,
    \Countable,
    \Serializable
{
}

Arguments in method/function declarations should be correctly indented

Argument lists may be split across multiple lines, where each subsequent line is indented once. When doing so, the first item in the list should be on the next line, and there should be only one argument per line. When the argument list is split across multiple lines, the closing parenthesis and opening brace should be placed together on their own line with one space between them.

Noncompliant Code Example

namespace Vendor\Package;

class ClassName
{
    public function aVeryLongMethodName(ClassTypeHint $arg1,   // Noncompliant
        &$arg2, array $arg3 = []) {

        $noArgs_longVars = function () use ($longVar1,         // Noncompliant
            $longerVar2,
            $muchLongerVar3
            ) {
                // body
            };
    }
}

Arguments in method/function calls should be correctly indented

Argument lists MAY be split across multiple lines, where each subsequent line is indented once. When doing so, the first item in the list MUST be on the next line, and there MUST be only one argument per line.

Noncompliant Code Example

<?php
$foo->bar($longArgument,    // Noncompliant
    $longerArgument,
    $muchLongerArgument);   // Noncompliant - closing parenthesis should be place on the next line

Compliant Solution

<?php
$foo->bar(
    $longArgument,
    $longerArgument,
    $muchLongerArgument
);

Closures declaration should be correctly spaced

Closures MUST be declared with a space after the function keyword, and a space before and after the use keyword.

Noncompliant Code Example

$closureWithArgsAndVars = function($arg1, $arg2)use   ($var1, $var2) {
    // body
};

Compliant Solution

$closureWithArgsAndVars = function ($arg1, $arg2) use ($var1, $var2) {
    // body
};

Class names, "extends" and "implements" keywords should be located on the same line

The goal of this coding rule is to make it obvious at first look that a class extends some other classes and/or implements some interfaces. The names of extended classes or implemented interfaces can be located on next lines.

Noncompliant Code Example

<?php
class ClassName
extends ParentClass {...}

Compliant Solution

<?php
class ClassName extends ParentClass {...}

or

<?php
class ClassName extends
ParentClass {...}
